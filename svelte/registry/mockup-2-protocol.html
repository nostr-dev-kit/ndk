Deep Protocol Integration

We don't just render data‚Äîwe understand Nostr at the protocol level. NIPs, event kinds, relay management, and signing are first-class concepts.

1. Event Kind Awareness

Components automatically adapt based on event kind. Pass a kind 1 (note), kind 30023 (article), or kind 9802 (highlight) and get the correct layout, metadata display, and interaction patterns.

// Same component, different kinds <EventCard event={noteEvent} /> <EventCard event={articleEvent} /> <EventCard event={highlightEvent} /> // Renders correctly for each type

2. NIP Implementation Support

Built-in support for NIP-05 verification badges, NIP-57 zaps, NIP-25 reactions, NIP-65 relay lists, and more. Components show verification status, handle relay hints, and respect event tags automatically.

// NIP-05 verification auto-displayed <User.Name user={user} /> <User.Nip05 user={user} /> // NIP-57 zaps with splits <ZapButton target={user} splits={[...]} />

3. Relay Management

Components understand relay pools, handle relay hints from 'e' and 'p' tags, and let users select preferred relays. Built-in relay health indicators and connection status.

// Relay selector with hints <RelaySelector relays={userRelays} hints={eventRelayHints} />

4. Negentropy Sync

First-class support for Negentropy set-reconciliation protocol. Efficiently sync large datasets from relays with animated progress indicators and smart batching.

// Efficient data sync visualization const sync = createNegentropySyncManager({ filter: { kinds: [1], authors: [...] } }); <NegentropySyncAnimated manager={sync} />
