The Three Runes You Need

$state

Reactive State

Declare reactive variables that trigger updates when changed. No magic assignment tracking—explicit and predictable.

let count = $state(0); count++; // triggers reactivity
$derived

Computed Values

Create values that automatically recompute when their dependencies change. Like computed properties, but better.

let doubled = $derived( count * 2 ); // updates when count changes
$effect

Side Effects

Run code in response to state changes. Perfect for subscriptions, logging, or any imperative logic.

$effect(() => { console.log(count); // runs when count changes });

The Builder Pattern

All 25 builders use Svelte 5 runes to create reactive, composable state managers. No Redux. No Pinia. Just clean, reactive Svelte code.

Example: createFollowAction

// Builder creates reactive follow state export function createFollowAction(config) { // Reactive target (user or hashtag) let target = $derived(config().target); // Check if already following (from NDK) let isFollowing = $derived( ndk.$follows.has(target.pubkey) ); // Execute follow/unfollow function execute() { if (isFollowing) { ndk.unfollow(target); } else { ndk.follow(target); } } return { target, isFollowing, execute }; } // Use in component const followAction = createFollowAction(() => ({ target: user })); // Reactive everywhere $: buttonText = followAction.isFollowing ? 'Unfollow' : 'Follow';
