Relay Card

A versatile card component for displaying Nostr relay information

The Relay Card component provides multiple ready-made variants for displaying relay metadata, status, and actions. Built using composable headless primitives, it supports NIP-11 metadata fetching, connection status indicators, and interactive relay management.

Basic Usage #

Example
<script> import { Relay } from '@nostr-svelte/ui'; import NDK from '@nostr-dev-kit/ndk'; const ndk = new NDK(); const url = "wss://relay.damus.io"; </script> <Relay.Root {ndk} {url}> <Relay.Icon size={80} /> <Relay.Header showUrl={true} /> <Relay.Metadata /> <Relay.Description /> </Relay.Root>

Primitives #

Relay.Root
Since v1.0.0

Root container component that provides relay context and manages state for all child primitives. Must wrap all other Relay primitives.

Props

Name Type Default Description
ndk required NDK - NDK instance for fetching relay metadata
url required string - WebSocket URL of the relay
autoFetch boolean true Automatically fetch NIP-11 metadata on mount
onLoad (metadata: RelayMetadata) => void - Callback fired when metadata loads
Relay.Icon
Since v1.0.0

Displays relay icon or favicon with automatic fallback handling and loading states.

Props

Name Type Default Description
size number 64 Icon size in pixels
fallback string - Fallback icon URL if relay icon fails to load
class string - Additional CSS classes to apply
Relay.Header
Since v1.0.0

Renders relay name and URL with optional link handling and metadata display.

Props

Name Type Default Description
showUrl boolean false Display the relay WebSocket URL
truncate boolean false Truncate long URLs with ellipsis
linkable boolean false Make the URL a clickable link
