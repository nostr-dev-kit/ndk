Proposed Component Structure

Component Name: UserHeader or UserProfile

Variants:

  • full - Complete header with banner, large avatar, bio, stats, and actions (Twitter/X style)
  • compact - Medium-sized header without banner, good for modals/cards
  • minimal - Small header for event cards and lists
  • horizontal - Single-line layout for sidebars and dropdowns
  • vertical - Centered card layout for user discovery

Key Features:

  • Display user avatar (with optional banner)
  • Show display name, NIP-05, and pubkey preview
  • Optional bio/about section with rich text support
  • Optional stats (notes count, following, followers)
  • Action buttons (Follow/Unfollow, Edit Profile, Share, More options)
  • Optional "Invited by" badge
  • Optional verification badge
  • Website/social links
  • Fully composable with Root + Children pattern

Composable Parts:

  • UserHeader.Root - Container with user data fetching
  • UserHeader.Banner - Cover image with fallback gradient
  • UserHeader.Avatar - Profile picture with size variants
  • UserHeader.Name - Display name with optional verification badge
  • UserHeader.Identifier - NIP-05 or pubkey display
  • UserHeader.Bio - About/bio section with rich content
  • UserHeader.Stats - Notes, following, followers counts
  • UserHeader.Actions - Follow button and action menu
  • UserHeader.Badges - Verification, invited by, etc.
  • UserHeader.Links - Website and social links
