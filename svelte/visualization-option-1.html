Layer 1: Chrome
The Container & Metadata
The chrome provides the consistent frame around all events: author info, avatar, timestamp, and action buttons. This stays the same regardless of what content is inside.

Layer 2: Content
The Event Body
Content components parse and render the actual event body. They handle markdown, mentions, hashtags, links, and media. Different event kinds use different content renderers.

Layer 3: Embeds
Referenced Events
When content references other events (via note1, nevent1, naddr1), those events are automatically fetched and rendered as rich previews inside the content. Different event kinds have different embed styles.
